# v0.3.0 manual QA — release checklist

For HUM-1419. These are the checks automation can't cover — mostly things a human has to
eyeball (visual rendering, real OAuth prompts, real double-click timing). The automated suite
(`pnpm test`, `pnpm e2e`, `pnpm e2e:real`) already covers correctness of sync logic, message
routing, and DOM injection under mocked/scripted conditions — see `docs/E2E_TESTING.md` and
`docs/qa/HUM-1281-real-e2e-harness.md`. Don't re-run those TC numbers (TC-001…TC-028) by hand;
this list only covers gaps.

## 0. Install — both browsers

The extension must load and run identically on Chrome and Arc (Arc has no native
`chrome.sidePanel`, which is moot now that the side panel is gone, but Arc's extension
sandboxing has tripped up MV3 behavior before — verify explicitly).

```bash
pnpm install
pnpm e2e:build   # builds apps/extension/.output/chrome-mv3
```

**Chrome:**
1. `chrome://extensions` → enable Developer mode → **Load unpacked** → select `apps/extension/.output/chrome-mv3`.
2. Confirm the dorv icon appears in the toolbar and the extension loads with no console errors (service worker "Inspect views").

**Arc:**
1. `arc://extensions` → enable Developer mode → **Load unpacked** → same directory.
2. Repeat TC-A2–TC-A5 below in Arc; confirm no Arc-specific rendering glitches (little/mini window mode, split view).

- [ ] **TC-A1** — Extension loads with no errors in both Chrome and Arc.

## 1. Onboarding

- [ ] **TC-A2** — Fresh install → open a markdown PR on GitHub → buttons show "needs setup" state → clicking opens the options page.
- [ ] **TC-A3** — Options page: enter a GitHub PAT missing required scopes → clear inline error, no silent failure.
- [ ] **TC-A4** — Options page: complete Google OAuth (`chrome.identity` real interactive flow, not the test harness's patched token) → profile name/email/avatar renders correctly.
- [ ] **TC-A5** — After both credentials saved, re-open the PR page → buttons transition out of "needs setup" without a manual reload.

## 2. Native button injection (replaces the removed side panel)

- [ ] **TC-A6** — Open a PR with multiple markdown files, files changed tab collapsed by default (GitHub lazy-loads diffs) → scroll to trigger lazy load → confirm a button appears next to *every* markdown file, not just the first.
- [ ] **TC-A7** — Open a PR with zero markdown files → confirm no buttons render anywhere (no orphaned shadow host).
- [ ] **TC-A8** — GitHub filename with a zero-width space (copy-pasted title) → button still renders and doc creation succeeds against the correct file.
- [ ] **TC-A9** — Google Docs comment card buttons: open a synced doc, confirm the push-to-GitHub icon button renders correctly inside the native GDoc comment card chrome (not just in isolation — check it doesn't clip/overlap GDoc's own UI).
- [ ] **TC-A10** — Resize the browser window / zoom to 150% → buttons on both GH and GDoc surfaces stay usable (no overflow clipping).

## 3. User-triggered sync (no more background alarm)

- [ ] **TC-A11** — Confirm sync never fires on its own — leave a linked PR/doc open and idle for 5+ minutes, verify no network activity in the service worker's Network tab until a button is clicked.
- [ ] **TC-A12** — Click "Create Doc" on a multi-markdown-file PR → confirm one Google Doc is created *per file*, not one doc with all files concatenated.
- [ ] **TC-A13** — Push a GitHub review comment to Google Docs, then push the *same* comment again (double-click, or click twice quickly) → confirm only one Drive comment exists (dedup-on-remote, the fix for the 1000-duplicate P0 — this is the single highest-risk regression to verify by hand).
- [ ] **TC-A14** — Resolve a GitHub review thread → click sync → confirm the mapped GDoc comment resolves; do the reverse (resolve in GDoc → push to GitHub).

## 4. Cross-session / real-world edge cases

- [ ] **TC-A15** — Close and reopen the browser mid-sync (kill the tab while a push is in flight) → reopen the PR → confirm no duplicate comment was created and the mapping is consistent.
- [ ] **TC-A16** — Revoke the GitHub PAT externally (delete it on github.com) → click sync → confirm a clear auth-error state, not a silent hang or crash.
- [ ] **TC-A17** — Let the Google OAuth token expire (or manually invalidate) → click sync → confirm a clear re-auth prompt, no stuck spinner.
