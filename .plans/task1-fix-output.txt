HUM-1415 Task 1: Fix nested reply sync loop — COMPLETE

What was changed:
- apps/extension/lib/adapters/direct.ts: In runPRSync(), the "GH replies → Doc" loop
  previously called mappingStore.getByGH(reply.inReplyToId) which only indexes
  CommentMappings (root comments). When the parent is a reply (nested reply-to-reply),
  this returned undefined and the nested reply was silently skipped.
  
  Fix: When mappingStore.getByGH() returns undefined, fall back to 
  replyMappingStore.getByGH() which indexes ReplyMappings. If found,
  build a synthetic CommentMapping using the parent ReplyMapping's
  docParentCommentId and docId.
